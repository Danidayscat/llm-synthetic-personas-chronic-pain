# Analysis Code — *[Evaluating LLM-Generated Synthetic Personas for Chronic Pain Research: Incremental Conditioning Improves Structural Fidelity but Fails Out-of-Sample Calibration]*

[![DOI](https://zenodo.org/badge/1228402164.svg)](https://doi.org/10.5281/zenodo.20017596)

## Overview

This repository contains the analysis code accompanying the manuscript:

> **Evaluating LLM-Generated Synthetic Personas for Chronic Pain Research: Incremental Conditioning Improves Structural Fidelity but Fails Out-of-Sample Calibration**
> Daniela Díaz-Catalán*, Jacopo Amidei, Andreas Kaltenbrunner, Jose Gregorio Ferreira De Sá, Juan V. Luciano, Rubén Nieto*
. *Artificial Intelligence in Medicine*, [2026].
> DOI: [paper DOI once available]

The code implements a comparative analysis of real participant (RP) data and
LLM-generated synthetic populations (GPT, DeepSeek, Llama) across 8 progressive
tasks, evaluating psychological outcomes (DASS-21, MI-RSWB-18) in a chronic
pain sample.

## Repository Contents

| File | Description |

## Data Availability

The `clean/` data directory is **not included** in this repository. The real-participant
dataset contains sensitive clinical information and will be shared upon reasonable request
to the corresponding author, subject to ethical approval.

Synthetic datasets generated by GPT, DeepSeek, and Llama are reserved for
forthcoming publications and are not released at this time.

To run the code, place your own datasets in a `clean/` subdirectory following the
variable naming conventions documented in Block 1 of the notebook.

## Requirements

- Python 3.10+

Open the notebook in JupyterLab or VS Code and run blocks sequentially.
